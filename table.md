Основние этапы SDLC

1. Выяснение требований - определяются цели тестирования, какие понадобятся ресурсы, сроки реализации проекта и бюджет
2. Анализ требований - сбор и анализ тестирования, уже здесь тестировщики включаются в работу, проверяют полноту требований, учтены все ли граничные случаи, какие тесты понадобятся, нет ли двусмысленностей, закладывается основа для будующих тест кейсов.
3. Проектирование - оценка архитектурных решений и сложность их проверки, предлагаются улучшения, разработка тестовой документации
4. Реализация - проводится модульное тестирование, написание тест кейсов,
5. Тестирование - происходит полный процесс тестирования всей системы, взаимодействия модулей между собой, приемочное тестирование с участием заказчика или пользователей
6. Релиз - запуск проекта в экслуатацию
7. Поддержка - дополняются тест кейсы и тестовая документация, проволится тестирование выявленных багов в процессе эксплуатации и новых функций при обновлении ПО
8. Ввод из эксплуатации 


В процессе Agile разработки тестирование идет параллельно с разработкой в рамках каждой итерации. Тестировщики учавствуют в планировании спринтов и ежедневных обсуждениях.


| Этапы                    | Роль                                                                                                                                                                                                                                                                                                                               |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Sprint Planning**      | анализ требований из backlog, определяются критерии приемки, определение нужных тестов, какие из них могут быть автоматизированы                                                                                                                                                                                                   |
| **Sprint**               | активное тестирование совместно с разработкий, написание тест-кейсов и чек-листов для нового функционала. Проводится функциональное и регрессионное тестирование, исследуется продукт, ищут скрытые дефекты, так же здесь происходит автоматизации тестирования и пишется код для повторяющихся тестов которые потом интегрируются |
| **Daily Scrum**          | ежедневное короткое совещание на котором обсуждается что было сделано за текущий день, что планируется сделать                                                                                                                                                                                                                     |
| **Sprint Review**        | в конце спринта тестировщиками проверяется готов ли спринт к стейкхолдерам, все ли тесты выполнены.                                                                                                                                                                                                                                |
| **Sprint Retrospective** | участвуют в анализе, смотрят что пошло хорошо а какие процессы можно улучшить.                                                                                                                                                                                                                                                     |
