Основние этапы SDLC

1. Выяснение требований
2. Анализ требований
3. Проектирование
4. Реализация
5. Контроль
6. Тестирование
7. Релиз
8. Поддержка
9. Ввод из эксплуатации 



| Этапы                | Что делают                                                                                                          | Роли                                 |
| -------------------- | ------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| Выяснение требований |                                                                                                                     | QA Manager                           |
| Анализ требований    | Определяет что будет проверяться, требования к тестированию, создается основа для тест-кейсов                       | QA Manager                           |
| Проектирование       | разработка тестовой документации и тест кейсов, предложения по улучшению продукта, определение методов тестирования | SDET, QA Manager                     |
| Реализация           | работа над тестовой докумнтацией, рекомендации для разработчиков                                                    | Test Automation Engineer             |
| Контроль             | выявление дефектов и передача их для исправления, повторное тестирование                                            | QA Engineer,Test Automation Engineer |
| Релиз                | проверка перед релизом и оценка готовности продукта                                                                 | SDET,Test Automation Engineer,       |
| Поддержка            | обновление документации, выявление дефектов готового продукта, рекомендации по разработки                           | SDET,Test Automation Engineer,       |
